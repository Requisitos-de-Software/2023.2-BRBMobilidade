# **Backward From**

## **Introdução**

O propósito deste documento é esclarecer de maneira transparente a rastreabilidade de todos os requisitos identificados ao longo do desenvolvimento do projeto. Para isso, os requisitos serão associados aos métodos específicos de obtenção e modelagem, seguindo a abordagem de rastreabilidade Backward From.

## **Objetivo**

Este documento tem como objetivo principal estabelecer de maneira clara e sistemática a conexão entre os requisitos identificados no decorrer do projeto e os métodos de elicitação e modelagem utilizados. Utilizando a abordagem de rastreabilidade Backward From, buscamos traçar uma linha transparente que vincule cada requisito aos seus métodos correspondentes, proporcionando uma compreensão abrangente da origem e do processo de desenvolvimento de cada componente essencial do projeto.

## **Metodologia**

Os requisitos funcionais e não funcionais previamente [Elicitados](https://requisitos-de-software.github.io/2023.2-BRBMobilidade/Elicitação/priorização%20de%20requisitos/Three%20Level%20Scale/) foram estruturados para criar a tabela a seguir. Nesse processo, verificou-se a conexão entre os requisitos e os artefatos associados a cada um, garantindo uma análise abrangente dos vínculos existentes.
